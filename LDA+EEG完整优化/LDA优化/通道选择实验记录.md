# LDA 通道选择实验记录

本文记录固定 `0-500 ms` 时间窗口下的通道选择实验。

## 实验口径

- 时间窗口固定为 `0-500 ms`。
- LDA 参数固定为前面 `0-500 ms` 的最佳配置：`bins=12`、`shrinkage=0.03`、类别先验 `[0.5, 0.5]`。
- 验证方式：训练集 12 个字符 trial 做 leave-one-character-out，按完整字符 trial 分组。
- 全通道作为 baseline。
- 通道评分只基于训练数据中 target 与 non-target 的差异，不使用测试答案选择通道。
- 测试参考列使用真实答案，仅用于结果记录，不作为严格盲选选择依据。

## 通道评分方法

在 `0-500 ms` 窗口内，对每个通道计算 target 与 non-target 平均 ERP 的差异，并用 pooled standard deviation 做标准化，得到近似 Cohen-d 的通道判别强度：

```text
score(channel) = mean_time(abs(mean_target - mean_nontarget) / pooled_std)
```

Leave-one-character-out 验证时，每一折只用该折训练部分计算通道排序；最终测试参考预测则用完整训练集计算通道排序。

## 结论

严格按训练集 leave-one-character-out 选择，推荐：

```text
0-500 ms + Top-12 channels + bins=12 + LDA shrinkage=0.03
```

Top-12 通道与全通道 baseline 的训练字符正确数同为 `10/12`，但 Top-12 的 PR-AUC 和 balanced accuracy 更高，同时特征维度从 `240` 降到 `144`，说明去掉低判别通道后模型更紧凑。

加权通道在测试参考上达到 `7/8`，但训练分组验证只有 `9/12`，低于 baseline 和 Top-12，因此不能作为严格盲选首选；可以作为答案揭示后的参考方案记录。

Top-K 过小会明显丢失信息。Top-4、Top-6、Top-8 都只有 `7/12`，说明虽然前几个通道判别强，但 P300 字符解码仍需要较宽的空间覆盖。Top-12 是当前实验中比较均衡的选择。

## 分析与建议

通道选择这部分的核心判断是：首选 Top-12，不建议直接使用全通道，也不建议过激进地只保留 Top-4、Top-6 或 Top-8。

Top-12 与全通道 baseline 的训练字符正确数都是 `10/12`，但 Top-12 的 PR-AUC 从 `0.486` 提升到 `0.491`，balanced accuracy 从 `0.748` 提升到 `0.759`。这说明后 8 个通道没有提供足够稳定的有效信息，反而可能增加维度和噪声。在当前只有 12 个独立训练字符 trial 的条件下，LDA 对高维特征较敏感，降低无效通道带来的维度负担是合理的。

Top-12 的另一个优势是模型更紧凑。全通道特征维度为 `240`，Top-12 降到 `144`，减少约 40% 特征维度，但没有损失字符正确数。这种结果比单纯追求更高维度更可靠，尤其适合小样本 P300 Speller。

过小的 Top-K 不适合作为主方案。Top-4、Top-6、Top-8 都只有 `7/12`，Top-10 也只有 `8/12`。这说明 P300 字符解码不是只靠少数几个最强通道即可完成；前几个通道虽然判别强，但行列聚合仍需要一定空间覆盖和互补信息。由于当前 20 个通道是匿名通道，没有真实电极位置，过度删通道会更容易误删有互补贡献的通道。

加权通道方案值得保留为补充分析，但不建议作为严格主方案。加权通道在测试参考中达到 `7/8`，但训练集分组验证只有 `9/12`，低于 Top-12 和全通道 baseline。严格实验报告中应以训练分组验证为选择依据，因此加权方案只能写成“测试参考表现较好，但训练验证不占优”。

最终建议写法：

> 在固定 `0-500 ms` 窗口下，按训练集 target/non-target 标准化 ERP 差异对通道排序。结果显示 Top-12 通道在保持 `10/12` 字符正确数的同时，将特征维度从 240 降至 144，并提升 PR-AUC 与 balanced accuracy。因此后续 LDA 可优先采用 Top-12 通道。过少通道会损失空间互补信息，加权通道虽在测试参考中表现较好，但训练分组验证不占优，故仅作为补充分析。
