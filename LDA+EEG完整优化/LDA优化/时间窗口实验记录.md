# LDA 时间窗口实验记录

本文记录本次对话中关于 P300 刺激后时间窗口选择的全部结果。所有窗口均在刺激后 `0-800 ms` 总范围内取子区间。

## 实验口径

- 数据：沿用交接版主预处理产物，`0.5-30 Hz`、`125 Hz`、`[-200, 0) ms` 基线校正。
- 模型：LDA，`solver="lsqr"`，类别先验固定为 `[0.5, 0.5]`。
- 验证：训练集 12 个字符 trial 做 leave-one-character-out，按完整字符 trial 分组。
- 解码：epoch 得分先按 trial 内 `event_id` 聚合，再分别选最高行和最高列映射字符。
- 选择主依据：训练集分组外推字符正确数，其次看 PR-AUC 与 balanced accuracy。
- 测试答案：真实答案揭示后只作为参考列记录，不作为严格盲选窗口选择依据。

## 结论

严格按训练集 leave-one-character-out 选择：

- 第一梯队：`0-600 ms` 与 `0-500 ms`，均为 `11/12 = 0.917`。
- `0-600 ms` 的 PR-AUC 更高：`0.496` vs `0.484`。
- `0-500 ms` 的测试参考更好：`6/8` vs `5/8`，且特征维度更低：`160` vs `200`。

综合训练分组验证、P300 生理解释、维度控制和测试参考，建议主报告采用：

```text
0-500 ms + 8 bins + LDA shrinkage=auto
```

## 为什么不是只用 300 ms 左右

P300 不是只在刺激后 300 ms 的单点出现。P300 Speller 的字符解码依赖每个行/列刺激的整段 ERP 形态，LDA 使用的是多通道时间分箱特征，而不是单个峰值。

`0-100 ms` 本身不应解释为 P300，但它可能提供以下信息：

- 刺激锁定后的早期视觉诱发响应；
- target 与 non-target 的早期注意调制差异；
- 波形对齐与整体形态上下文；
- 与后续 N2/P3 形态共同形成的判别模式。

实验结果也支持这一点：只保留更“纯 P300”的后段窗口，如 `200-500 ms`、`250-550 ms`、`300-600 ms`，训练字符正确数分别只有 `7/12`、`5/12`、`3/12`。这说明本数据的有效判别信息不只在 300 ms 附近。

另一方面，窗口过长也会引入晚期噪声。`0-800 ms` 只有 `10/12`，`0-600 ms` 虽然训练为 `11/12`，但测试参考下降到 `5/8`。因此 `0-500 ms` 是当前数据里更均衡的选择：保留早期视觉/注意信息和主要 P300 上升、峰值、下降段，同时避免过多晚期波动。
