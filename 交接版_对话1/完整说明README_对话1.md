# P300 Speller 实验交接说明

## 1. 交付范围

本文件夹是本轮实验的最小交接包，仅包含：

- `P300完整实验主线交接版_对话1.ipynb`：可执行主Notebook；
- `完整说明README_对话1.md`：完整方法、依据、结果和交接说明；
- `精简主线README_对话1.md`：快速复现与主结论。

EEGNet实验未纳入本交接包。主Notebook只比较传统机器学习模型。

## 2. 实验任务与数据

实验为6×6 P300 Speller。每次行/列闪烁形成一个EEG epoch，分类目标为判断该刺激是否包含P300；随后在每个字符trial内分别聚合行事件和列事件得分，选择最高分行列并映射为字符。

- 单被试S1；
- 20个匿名EEG通道；
- 原始采样率250 Hz；
- 12个已知训练字符trial；
- 每字符5轮、每轮12个有效刺激；
- 共720个训练epoch，其中120个target、600个non-target；
- 8个未知测试trial不参与本阶段选择。

原始数据应位于仓库根目录的`P300-S1/`。建议从仓库根目录启动Jupyter，以保证相对路径正确。

### 3.2 预处理

固定带通为四阶Butterworth零相位`0.5–30 Hz`。先对完整连续trial滤波，再提取`[-200,800) ms` epoch，以减少逐epoch滤波边缘效应。

四个方案采用采样率与基线窗口的二因素组合：

| 方案 | 采样率 | 基线 |
|---|---:|---:|
| 主方案 | 125 Hz | `[-200,0) ms` |
| 备选1 | 250 Hz | `[-200,0) ms` |
| 备选2 | 125 Hz | `[-100,0) ms` |
| 备选3 | 250 Hz | `[-100,0) ms` |

选择主方案的理由：

- P300属于低频ERP，125 Hz仍可覆盖0.5–30 Hz有效频带；
- 多相抗混叠降采样降低后续特征维度和计算量；
- `[-200,0) ms`提供完整刺激前基线；
- 备选方案保留用于后续消融，而不是依据未知测试答案挑选。

主方案处理顺序为：`0.5–30 Hz带通 → epoch提取 → 250→125 Hz降采样 → [-200,0) ms基线校正`。

### 3.3 P300特征识别区间

在Shrinkage LDA下比较：

- 参考：`0–800 ms`；
- 候选：`0–480 ms`；
- 候选：`100–600 ms`；
- 候选：`200–400 ms`。

每个区间构造约25 ms的通道均值特征。外层使用Leave-One-Character-Out，内层GroupKFold选择LDA收缩系数。

| 区间 | 字符准确率 | PR-AUC | ROC-AUC | 平衡准确率 |
|---|---:|---:|---:|---:|
| 100–600 ms | 0.750 | 0.445 | 0.761 | 0.714 |
| 0–800 ms | 0.667 | 0.438 | 0.750 | 0.698 |
| 0–480 ms | 0.583 | 0.451 | 0.773 | 0.706 |
| 200–400 ms | 0.417 | 0.316 | 0.679 | 0.628 |

按预先规定的“字符准确率优先，其次PR-AUC和平衡准确率”，选择`100–600 ms`作为后续特征识别区间。0–480 ms的epoch级AUC较高，但字符级行列聚合较差，说明epoch指标不能替代最终字符指标。

### 3.4 传统模型候选

候选模型：

- Shrinkage LDA；
- Logistic Regression；
- Linear SVM；
- xDAWN + LDA；
- xDAWN + Riemannian Tangent Space。

文献依据：

- Krusienski等对P300 Speller分类方法的比较支持线性判别与线性SVM作为实用基线：[PubMed](https://pubmed.ncbi.nlm.nih.gov/17124334/)；
- xDAWN用于增强P300诱发响应：[PubMed](https://pubmed.ncbi.nlm.nih.gov/19174332/)；
- ERP协方差的黎曼几何方法适合P300/ERP分类：[论文](https://arxiv.org/abs/1409.0107)。

选择传统线性模型而不是复杂非线性模型的主要原因是：训练数据只有12个独立字符trial；同一trial内epoch存在时序与重叠相关；线性、带正则化的方法更适合当前小样本并具有更强可解释性。

## 4. 验证与防泄漏设计

- 分组单位是完整字符trial，不随机拆分epoch；
- 外层Leave-One-Character-Out用于估计泛化性能；
- 普通线性模型的超参数仅在外层训练字符内用GroupKFold选择；
- 预设选择顺序：字符准确率 → PR-AUC → balanced accuracy；
- 8个未知测试trial不参与预处理、时间窗、模型或参数选择。

## 5. 传统模型结果

| 模型 | 字符准确率 | PR-AUC | ROC-AUC | 平衡准确率 |
|---|---:|---:|---:|---:|
| Shrinkage LDA | 8/12 = 0.667 | 0.446 | 0.753 | 0.695 |
| Linear SVM | 8/12 = 0.667 | 0.429 | 0.737 | 0.678 |
| Logistic Regression | 7/12 = 0.583 | 0.382 | 0.726 | 0.669 |
| xDAWN + LDA | 5/12 = 0.417 | 0.406 | 0.738 | 0.686 |
| xDAWN + Riemann TS | 4/12 = 0.333 | 0.401 | 0.727 | 0.657 |

Shrinkage LDA与Linear SVM字符准确率并列，但Shrinkage LDA的PR-AUC和平衡准确率更高，因此选择Shrinkage LDA作为下一阶段基础模型。

## 6. 当前主线结论

- 主预处理：`0.5–30 Hz + 125 Hz + [-200,0) ms baseline`；
- 特征区间：`100–600 ms`；
- 特征形式：约25 ms时间窗的20通道均值；
- 基础模型：Shrinkage LDA；
- 主要评价：字符准确率；
- 当前分组OOF字符表现：基础模型8/12；窗口实验中100–600 ms达到9/12；
- 未知测试集未用于选择。

基础模型表与窗口表来自两个独立嵌套实验模块，后续同学应在统一的完整嵌套管线中把`100–600 ms`和模型超参数联合评估，不能简单把两个模块的最好数字拼接为最终性能。

## 7. 后续建议

下一阶段建议保持外层trial分组不变，依次研究：

1. 在完整嵌套验证内联合优化时间窗和LDA收缩；
2. 比较20/25/40/50 ms时间分辨率；
3. 匿名通道子集选择必须在内层完成；
4. 比较类别先验、分数校准和稳健标准化；
5. 优化行列聚合，包括轮次加权、分数归一化和置信边际；
6. 做1–5轮刺激消融；
7. 在所有规则锁定后再对未知测试trial生成一次预测。
